# Contributing to CODESAGE

Thank you for your interest in contributing to CODESAGE! This document provides guidelines for contributors.

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/codesage.git
   cd codesage
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
4. Create a new branch for your feature:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Workflow

### Running Tests

```bash
npm test
```

### Building the Project

```bash
npm run build
```

### Running in Development Mode

```bash
npm run dev
```

## Code Style

- Use TypeScript for all new code
- Follow the existing code style
- Add comments for complex logic
- Include tests for new features

## Submitting Changes

1. Ensure all tests pass
2. Build the project successfully
3. Commit your changes with a clear message
4. Push to your fork
5. Create a pull request

## Pull Request Process

- Describe your changes clearly
- Link any relevant issues
- Ensure your PR passes all checks
- Wait for code review

## Bug Reports

When reporting bugs, please include:
- Node.js version
- Operating system
- Steps to reproduce
- Expected behavior
- Actual behavior

## Feature Requests

Feel free to open issues for feature requests. Please:
- Describe the use case
- Explain why it would be useful
- Consider if it fits the project's goals

## Questions

Join our discussions or open an issue for questions.
